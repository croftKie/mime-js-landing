MimeJS - Templating Framework

MimeJS

MimeJS is a component based framework that uses vanilla JavaScript and HTML Templating. Think React, without the JSX.

Set up the file structure

src

templates

components

public

assets

css

js

MimeJS uses a specific folder structure to create your application. In the parent folder, create the above file structure.

Install (Option 1)

You can install MimeJS to a NodeJS project

NPM init

run the above command in the terminal of your project folder

npm install mime-js

Install the framework with the above command

"node ./node_modules/mime-js"

Add this command to a "build" script in your package.json file.

Install (Option 2)

Access MimeJS through NPX

NPM init

run the above command in the terminal of your project folder

"npx mime-js"

Add this command to a "build" script in your package.json file.

Visit the docs for best practices

For advanced uses of MimeJS check out the docs to learn how to include third-party code in your project build, how to configure CSS/JS and Assets and how to handle naming conventions for templates and components.

Release Version 1.0

The first public build of MimeJS, it should be seen as a live-beta that is actively being improved upon, with your support and feedback.

Version 1.0 Documentation

For ease of use, I am currently writing a documentation page for MimeJS version 1.X to provide new users with an easy guide.

Nested Layout files

To allow for more use-cases, I am working on improving the layout functionality to handle nested components, rather than a single layer of components per layout file.

MimeJS CLI Tool

To improve the initialisation of MimeJS applications I'll be developing a CLI package for easy bootstrapping of a MimeJS file structure and build commands.

Improved performance features

Follow the release of all core functionality, and the CLI tool. I'll be looking to improve the minification of build files and asset files to improve the size of final build folders for larger projects.

Version 2.0 release

With the creation of the nested layout feature, the sister CLI tool and improved performance / bug squashing, I'll be moving the framework from its 1.0 state to 2.0, denoting full and stable release.
